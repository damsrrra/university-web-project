HTML Techniques
The code begins with the DOCTYPE html declaration, which specifies that the document follows the HTML5 standard.The head section contains the document's metadata, including the title,The body is divided into three main sections Header, Main Content and Footer. The navigation bar is created using the 'nav' element, which groups together the links to different sections of the page.The profile section is structured using a 'div' container that holds the user's image, name, and etc.A progress bar is used to visually represent different skill level. The outer div acts as a container, while the inner div fills up dynamically based on the skill percentage. The experience and goals are presented in an unordered list to ensure better readability. In Availability Calendar a table is used to display availability dates, with special classes assigned to highlight selected days. The footer contains the logo and relevant links.

CSS Styling Techniques
The background color creates a professional appearance, while that the footer stays at the bottom of the page. In navigation links are neatly aligned to the right, improving usability.The border-radius (50%) makes the image circular, and margin: 0 auto; centers it, creating a clean and balanced layout. The progress bar technique is used to visually represent skill levels. The inner div dynamically adjusts its width to reflect the level of expertise. The hover effect (transform: scale(1.1);) adds an interactive touch by making dates stand out when hovered over.For Footer background: url Style background image is used to match the branding, and the width is set to 120% for a full-page effect.

Accessibility

Accessibility was prioritized by using semantic HTML elements and ARIA attributes. From the user profile users can personalize their information and they can mark available dates for meetings or events. Alse Sitemap has interactive nodes that represent each page or section of your website and Let users add custom labels or notes to specific sections of the sitemap, particularly useful for internal collaboration or understanding. Users can click on them to view more information or go directly to that sectionThe pages are also responsive, ensuring a good user experience on different devices.

Compliance

The website has been developed with various compliance features that align with Janet regulations, ensuring it meets accessibility, data protection, security, and content quality standards. Accessibility is achieved through the use of semantic HTML elements such as headeer, footer, and nav, along with descriptive alt text for images, making it easier for screen readers to navigate. Keyboard navigation is supported, and the site follows color contrast guidelines for users with low vision
